## 情报分享及其他要点

- **GTest 不支持 fork**: 研究确认 GTest 设计上不支持在 `TEST()` 体内 `fork()`——子进程复制 GTest 全局状态（测试注册、输出缓冲）导致崩溃。Linux 上 GTest 更推荐 `clone()`。对测试多进程代码的场景，建议使用 `EXPECT_DEATH`/重构为独立二进制/集成测试框架。
- **MUSA-Runtime vs linux-ddk/musa 关系**: 确认两者是独立项目。MUSA-Runtime (`libmusart.so`) 是 CUDA Runtime API 兼容层，linux-ddk/musa (`libmusa.so`) 是驱动实现。Runtime 通过 `muGetExportTable` 动态加载驱动函数指针。
- **musaMemAddressReserve 不存在于 MUSA-Runtime**: 虚拟内存管理 API (13 个 `muMem*`) 是驱动级别 API，与 CUDA 设计一致——`cuMemAddressReserve` 也没有 `cudaMemAddressReserve` 运行时包装。运行时通过 `muGetProcAddress` 可动态获取。
