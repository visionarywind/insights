# M1 最小白盒性能建模闭环实施清单

## 1. 范围

M1 只覆盖三类 API：

| API | 覆盖的软件成本 |
| --- | --- |
| `muLaunchKernel` / `musaLaunchKernel` | launch、dependency、queue、command build、submit |
| `muMemAlloc_v2` / `muMemFree_v2` / `musaMalloc` / `musaFree` | synchronous memory create/init、memory lookup/synchronize/destroy；virtual/pool free 分支覆盖 pool disable access |
| `muStreamSynchronize` / `musaStreamSynchronize` | stream lookup、last command、sync wait、error query |

M1 的目标不是覆盖 Top90 API，而是验证完整链路：

```text
source rule
  -> ModelEvent
  -> MUPTI collector
  -> relation builder
  -> api_cost_breakdown
  -> overhead report
```

| 交付物 | 内容 |
| --- | --- |
| `source_rules/muLaunchKernel.yaml` | launch 路径源码规则 |
| `source_rules/muMemAlloc.yaml` | allocation 路径源码规则 |
| `source_rules/muMemFree.yaml` | free 路径源码规则 |

## 3. Source Rule

### 3.1 `muLaunchKernel`

源码路径：

```text
Runtime musaLaunchKernel
  -> Driver muLaunchKernel wrapper
  -> Driver muapiLaunchKernel
  -> Core Stream::CmdLaunchKernel
  -> Context::ResolveDependencyAndQueueCommand
  -> Stream::QueueCommand
  -> Stream::AsyncSubmit
  -> DispatchCommand::Build
  -> DispatchCommand::Submit
  -> Command::SubmitToQueue
  -> HAL Queue::Submit
  -> M3D Queue::SubmitInternal
  -> DRM Queue::OsSubmit
```

成本项：

```text
T_launch =
  T_runtime_wrapper
+ T_driver_wrapper
+ T_function_lookup
+ T_stream_lookup
+ T_command_create
+ T_dependency_resolve
+ T_queue_command
+ T_command_build
+ T_merge_check
+ T_submit_to_queue
+ T_hal_m3d_submit
```

必须事件：

| 事件 | 类型 | 位置 |
| --- | --- | --- |
| `RUNTIME_API_SPAN` | span | Runtime wrapper enter/exit |
| `DRIVER_API_SPAN` | span | Driver wrapper enter/exit |
| `COMMAND_CREATE` | instant | `Stream::CmdLaunchKernel` |
| `DEPENDENCY_RESOLVE_SPAN` | span | `Context::ResolveDependencyAndQueueCommand` |
| `STREAM_QUEUE_COMMAND_SPAN` | span | `Stream::QueueCommand` |
| `COMMAND_BUILD_SPAN` | span | `DispatchCommand::Build` |
| `COMMAND_MERGE_DECISION` | instant/counter | `Stream::AsyncSubmit` |
| `COMMAND_SUBMIT_SPAN` | span | `Command::SubmitToQueue` |
| `HAL_M3D_SUBMIT_SPAN` | span | HAL/M3D submit 边界 |
| `API_COMMAND_RELATION` | relation | API correlation -> command |
| `COMMAND_SUBMISSION_RELATION` | relation | command -> submission |
| `SUBMISSION_ACTIVITY_RELATION` | relation | submission -> kernel activity |

### 3.2 `muMemAlloc_v2`

源码路径：

```text
Runtime musaMalloc
  -> Runtime musaapiMalloc
  -> Driver muMemAlloc_v2
  -> TlsCtxTop context lookup
  -> build MemoryCreateInfo(memoryTypeGeneral, virtual/device-mapped/sub-allocatable flags)
  -> Context::CreateMemory
  -> Memory::Init
  -> HAL memory allocate / map
```

成本项：

```text
T_alloc =
  T_runtime_wrapper
+ T_driver_wrapper
+ T_validation
+ T_context_lookup
+ T_memory_create_info
+ T_context_create_memory
+ T_memory_init
+ T_hal_alloc_or_map
+ T_error_check
```

必须事件：

| 事件 | 类型 | 位置 |
| --- | --- | --- |
| `MEM_ALLOC_API_SPAN` | span | API enter/exit |
| `MEM_ALLOC_VALIDATE_SPAN` | span | `muapiMemAlloc_v2` 参数检查 |
| `CONTEXT_LOOKUP_SPAN` | span | `TlsCtxTop` / context resolve |
| `MEMORY_CREATE_INFO_POINT` | instant | `MemoryCreateInfo` 构造完成 |
| `CONTEXT_CREATE_MEMORY_SPAN` | span | `Context::CreateMemory` |
| `MEMORY_INIT_SPAN` | span | `Memory::Init` |
| `HAL_ALLOC_SPAN` | span | HAL allocate/map |
| `API_MEMORY_RELATION` | relation | API -> allocation object |

注意：M1 的 `musaMalloc -> muMemAlloc_v2` 是同步 general memory 创建路径，不应按 memory pool hit/miss 建模。`MemoryPool::CreateMemory/ModifyAccess` 属于 async/pool allocation，放在 M2 memory pool 家族。

### 3.3 `muMemFree_v2`

源码路径：

```text
Runtime musaFree
  -> Runtime musaapiFree
  -> Driver muMemFree_v2
  -> Platform::GetMemoryByDevicePointer
  -> memory type / offset validation
  -> IMemory::Synchronize
  -> non-virtual memory: IContext::DestroyMemory
  -> virtual memory with pool: IStream::CmdMemFree(blocking=true)
       -> Stream::CmdMemFree
       -> Stream::AsyncMemFree
       -> MemoryPool::DisableAccess
       -> CallbackCommand deferred destroy
```

成本项：

```text
T_free =
  T_runtime_wrapper
+ T_driver_wrapper
+ T_memory_lookup
+ T_type_check
+ T_memory_synchronize
+ T_context_destroy_memory
+ T_pool_disable_access_virtual_branch
+ T_callback_queue_virtual_branch
```

必须事件：

| 事件 | 类型 | 位置 |
| --- | --- | --- |
| `MEM_FREE_API_SPAN` | span | API enter/exit |
| `MEM_FREE_LOOKUP_SPAN` | span | `Platform::GetMemoryByDevicePointer` |
| `MEM_FREE_TYPE_DECISION` | instant | memory type / offset decision |
| `MEMORY_SYNCHRONIZE_SPAN` | span | `IMemory::Synchronize` |
| `CONTEXT_DESTROY_MEMORY_SPAN` | span | non-virtual `Context::DestroyMemory` |
| `MEM_POOL_DISABLE_ACCESS_SPAN` | span | virtual/pool branch `MemoryPool::DisableAccess` |
| `MEM_FREE_CALLBACK_COMMAND_CREATE_SPAN` | span | virtual/pool branch callback command |
| `API_MEMORY_RELATION` | relation | API -> allocation object |

注意：普通 `musaMalloc -> musaFree` 路径通常是 general memory，释放主路径是 `Synchronize -> Context::DestroyMemory`；pool disable/callback 只覆盖 virtual memory with pool 分支。

### 3.4 `muStreamSynchronize`

源码路径：

```text
Runtime musaStreamSynchronize
  -> Driver muStreamSynchronize
  -> stream lookup
  -> Stream::WaitFinish
  -> Command::Wait 或 Stream::AsyncWait
  -> semaphore wait
  -> engine last error query
  -> user pool update
```

成本项：

```text
T_stream_sync =
  T_runtime_wrapper
+ T_driver_wrapper
+ T_stream_lookup
+ T_last_command_lookup
+ T_sync_wait
+ T_engine_error_query
+ T_user_pool_update
```

必须事件：

| 事件 | 类型 | 位置 |
| --- | --- | --- |
| `STREAM_SYNC_API_SPAN` | span | API enter/exit |
| `STREAM_LOOKUP_SPAN` | span | stream lookup |
| `STREAM_WAIT_FINISH_SPAN` | span | `Stream::WaitFinish` |
| `COMMAND_WAIT_SPAN` | span | command wait |
| `SEMAPHORE_WAIT_SPAN` | span | semaphore wait |
| `ENGINE_ERROR_QUERY_SPAN` | span | engine error query |
| `SYNC_WAIT_REASON` | instant | wait reason |
| `API_SYNC_RELATION` | relation | API -> waited command/activity |

M1 collector 至少输出：

```text

api_cost_breakdown.parquet

```

字段要求：

| 表 | 必须字段 |
| --- | --- |
| `api_events` | api id、name、domain、start/end、status、correlation id |
| `model_events` | event type、domain、event id、span id、parent id、payload |
| `activity_events` | kernel/memcpy/sync activity、start/end、stream、correlation |
| `relations` | src type、src id、dst type、dst id、confidence |
| `api_cost_breakdown` | api、total、runtime、driver、memory、queue、build、submit、sync、unknown |

## 6. Relation Builder

必须重建：

```text
Runtime API -> Driver API
Driver API -> command
command -> submission
submission -> kernel activity
sync API -> waited command/activity
memory API -> allocation object
```

M1 验收：

| 关系 | 标准 |
| --- | --- |
| launch API -> kernel activity | 召回率不低于 95% |
| stream sync -> waited command/activity | 能说明等待对象和等待时间 |
| alloc/free -> allocation object | 能关联 ptr、size、memory type、status；virtual/pool 分支额外关联 pool id |

## 7. Cost Breakdown

M1 输出示例：

| API | total_us | runtime_us | driver_us | memory_us | queue_us | build_us | submit_us | sync_wait_us | unknown_us |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| `muLaunchKernel` | 18.2 | 1.1 | 2.4 | 0.0 | 3.0 | 5.2 | 6.1 | 0.0 | 0.4 |
| `muMemAlloc_v2` | 74.6 | 0.8 | 2.2 | 41.5 | 0.0 | 0.0 | 27.4 | 0.0 | 2.7 |
| `muStreamSynchronize` | 310.0 | 0.7 | 1.9 | 0.0 | 0.0 | 0.0 | 0.0 | 304.8 | 2.6 |

unknown cost 必须输出。unknown 过高时，报告无效。

建议阈值：

```text
M1 三类 API unknown_us / total_us <= 15%
```

## 8. Overhead 验收

必须跑三种模式：

| 模式 | 说明 | 建议标准 |
| --- | --- | --- |
| trace off | 编译包含埋点但未开启 collector | <= 0.1% 或在噪声内 |
| API-only | 只采集 Runtime/Driver API | <= 1% |
| internal targeted | 采集 M1 ModelEvent | <= 3% |

必须报告：

```text
baseline_time
trace_off_time
api_only_time
internal_targeted_time
overhead_percent
dropped_records
flush_errors
buffer_overflow
```

## 9. M1 验收标准

M1 完成条件：

1. 三类 API 均有 source rule。
2. ModelEvent private hook 可采集 M1 事件。
3. collector 能输出 API、activity、ModelEvent、relation 四类数据。
4. launch 路径能重建 API -> command -> submission -> kernel。
5. stream sync 能说明等待对象和等待原因。
6. alloc/free 能说明 memory create/init、memory lookup/synchronize/destroy；virtual/pool 分支能说明 pool disable access。
7. `api_cost_breakdown` 能输出分项耗时。
8. relation recall 不低于 95%。
9. unknown cost 不高于 15%。
10. trace off/API-only/internal targeted 开销有报告。

## 10. 不进入 M1 的内容

以下内容不进入 M1，避免范围失控：

| 内容 | 说明 |
| --- | --- |
| 全量 Top90 API | M2 扩展 |
| CUDA/ROCm 对标工具实现 | 只保留设计参考 |
| PC sampling / thread trace | 硬件深度分析，不是 M1 软件模型核心 |
| 完整 graph 建模 | M2/M3 处理 |
| 多卡行为 | M2/M3 处理 |
| public MUPTI ABI | M1 使用 private hook，稳定后再升级 |
