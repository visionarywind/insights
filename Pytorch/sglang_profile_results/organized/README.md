# SGLang Profile Results Index

## Directory Layout

This directory is the cleaned entry point for the current SGLang/API profile data.

```text
organized/
├── reports/
├── derived/
├── raw/
│   ├── sglang_traces/
│   ├── timelines/
│   └── api_activity/
└── validation/
    └── api_timeline_min/
```

All duration fields in derived files use `us`.

## Recommended Reading Order

1. `reports/sglang_api_profile_comparison_us.md`
   - Main comparison report.
   - Summarizes SGLang profiler and API-side activity profiler results.

2. `reports/profile_comparison_methodology_us.md`
   - Explains how data was collected, converted, and compared.
   - Includes the unit conversion rule: SGLang trace `dur` is already `us`; API activity `total_ms` is converted to `total_us`.

3. `reports/api_profiler_data_check.md`
   - Explains why the API profiler summary looked sparse and where the full timeline data is stored.

4. `derived/*.tsv`
   - Machine-readable summaries for follow-up analysis.

5. `raw/`
   - Original trace/log/timeline data for audit or re-parsing.

## Derived Data

- `derived/sglang_trace_category_summary_us.tsv`
  - SGLang Chrome trace grouped by category, such as `kernel`, `privateuse1_runtime`, and `privateuse1_driver`.

- `derived/sglang_trace_gpu_api_summary_us.tsv`
  - SGLang GPU/API-like events grouped by event name.

- `derived/api_activity_summary_us.tsv`
  - API-side activity profiler summary grouped by API/kernel name.

- `derived/validation_summary.json`
  - Run status and trace file ordering metadata.

## Raw Data

- `raw/sglang_traces/*.trace.json.gz`
  - Original SGLang prefill/decode Chrome traces.

- `raw/timelines/sglang_raw_gpu_timeline_us.tsv`
  - SGLang GPU/API-like events sorted by Chrome trace `ts`.
  - This file preserves true execution order: prefill first, then decode.

- `raw/timelines/sglang_raw_gpu_timeline_head.md`
  - Human-readable preview of the raw timeline.

- `raw/api_activity/api_activity_console.log`
  - Original API-side activity profiler console output.

## API Profiler Timeline Check

- `api_profiler_timeline_check/gpu_activity_timeline.tsv`
  - SGLang API profiler timeline emitted by the modified activity profiler.

- `api_profiler_timeline_check/sglang_api_summary_us.tsv`
  - Summary converted from the SGLang API profiler timeline.

## Validation Data

- `validation/api_timeline_min/timeline.tsv`
  - Minimal MUSA workload timeline emitted by the modified API activity profiler.

- `validation/api_timeline_min/converted/validate_timeline_us.tsv`
  - Normalized timeline generated by `tools/convert_mupti_activity_timeline.py`.

- `validation/api_timeline_min/converted/validate_summary_us.tsv`
  - Summary grouped by type/domain/name from the minimal validation run.

## Related Scripts

- `tools/convert_mupti_activity_timeline.py`
  - Converts profiler timeline TSV to normalized timeline, summary, and preview files.

- `tools/merge_sglang_api_timelines.py`
  - Merges SGLang Chrome trace events and API activity timeline into one time-ordered TSV.

- `tools/run_sglang_gpu_activity_profile_musa.sh`
  - Runs SGLang with MUPTI activity profiling and writes `gpu_activity_timeline.tsv` when using the updated profiler.
