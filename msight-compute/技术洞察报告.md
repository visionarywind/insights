## 一、项目定位

**msight-compute** 是摩尔线程 GPU 的性能剖析（profiling）工具链，对标 **NVIDIA Nsight Compute**。版本 1.3.0。它通过 MUPTI（MooreThreads Profiling Tools Interface）采集 GPU kernel 运行时的性能指标、timeline 数据和 PC 采样信息，最终生成 `.mcu-rep` 报告文件并提供 CLI/GUI 两种交互方式。

## 三、模块架构

### 3.1 核心数据流

```
用户命令行/GUI
    │
    ▼
profiler (编排器)
    │  ┌─ profiler_host: 协调采集会话
    │  └─ profiler_target: 被 profile 的目标进程管理
    │
    ▼
capture (数据采集)
    │  ┌─ mc_capture: 采集会话管理
    │  ├─ mc_mupti_capture: MUPTI 回调注册
    │  └─ mc_app_replay_profiler: 应用 replay profiling
    │
    ▼
MUPTI (GPU 剖析接口)
    │  调用 libmupti.so
    │
    ▼
GPU Driver / KMD → 硬件性能计数器
```

### 3.2 关键模块详解

#### profiler —— 剖析编排器
- **profiler_host.cpp**: 主控端，负责创建/管理采集会话、配置 MUPTI 参数
- **profiler_target.cpp**: 目标进程管理，控制被 profile 的应用生命周期
- **profiler_types.h**: 公共类型定义（采集模式、replay 配置等）

#### capture —— MUPTI 数据采集
- **mc_capture.cpp/h**: 采集会话生命周期管理（初始化 → 订阅 → 启用 domain → 开始采集 → 结束）
- **mc_mupti_capture.cpp/h**: MUPTI 回调注册与数据处理
- **mc_app_replay_profiler.cpp/h**: 应用 replay 模式（单个 kernel 多次重放以获得精确指标）
- **mc_check_summary.cpp/h**: 采集结果的校验与摘要生成

#### injection —— 预加载注入
- **mupti_injection.cpp/h**: 预加载入口，通过 `LD_PRELOAD` 注入目标应用
- **mupti_injection_global_data.cpp/h**: 全局会话状态管理

#### mupti_api —— MUPTI API 封装层
- **mupti_api.h**: 模板化的 MUPTI 动态加载封装，延迟绑定所有 profiling API
- 涵盖 Activity API、Callback API、Profiler Session API、PC Sampling API
- 所有函数指针通过 `MUptiLoader` 动态加载，实现运行时绑定

#### report —— 报告 I/O
- **report/write/**: 将采集数据序列化为 `.mcu-rep` protobuf 格式
- **report/read/**: 解析 `.mcu-rep` 文件，供 GUI 展示和 CLI 导出

#### mt_rules —— Metrics 规则引擎
- **metrics_manager.cpp/h**: 从 YAML 规则文件加载 metrics 定义
- **action.cpp/h**: 基于 PFM 解析器的 metric 计算逻辑
- 支持自定义 metrics YAML 添加

### 3.3 两种 Profiling 模式

| 模式 | 说明 | 入口 |
|------|------|------|
| **Direct Profiling** | msight-compute 直接启动目标应用，通过 MUPTI 采集 | `mc_capture.cpp` |
| **Injection Profiling** | 通过 `LD_PRELOAD` 注入已运行的应用（仅 Linux） | `mupti_injection.cpp` |
| **Application Replay** | 采集 kernel launch 序列，然后逐一重放以获得精确度量 | `mc_app_replay_profiler.cpp` |

## 四、msight-compute / musa / MUSA-Runtime / MUPTI 协作关系

这四个组件的协作形成了一个完整的分层架构：**MUSA-Runtime** 是上层计算 API，**musa Driver** 是底层驱动接口，**MUPTI** 是剖析探针，**msight-compute** 是剖析消费端。

### 4.1 分层体系

```
┌──────────────────────────────────────────────────┐
│  Application (PyTorch, SGLang, .cu 程序)          │
│  调用: musaLaunchKernel / musaMemcpy / musaMalloc  │
└────────────────────┬─────────────────────────────┘
                     │
                     ▼
┌──────────────────────────────────────────────────┐
│ MUSA-Runtime  (libmusart.so  ~/MUSA-Runtime)     │
│  ├─ 340+ 导出符号，CUDA Runtime API 兼容层        │
│  ├─ Device / Stream / Event / Graph 管理          │
│  ├─ 通过 ExportTable 调用 Driver 层               │
│  └─ 封装 muInit / muCtxCreate / muLaunchKernel    │
└────────────────────┬─────────────────────────────┘
                     │ mu* Driver API
                     ▼
┌──────────────────────────────────────────────────┐
│ MUSA Driver  (linux-ddk/musa/, 560 muapi* 函数)   │
│  ├─ API 层 → Core 层 → HAL 层 → M3D 层            │
│  ├─ 内存管理: memMgr / MemoryPool Sub-Alloc       │
│  ├─ Green Context: SM 分区隔离 (muDevSmResource*) │
│  ├─ Peer 映射、DRM/WDDM2                          │
│  └─ ioctl → KMD (gr-kmd 内核驱动)                  │
└──────┬──────────────────────┬────────────────────┘
       │                      │
       ▼                      ▼
┌──────────────┐    ┌──────────────────────────────┐
│  KMD 内核驱动  │    │ MUPTI  (libmupti.so)          │
│  (gr-kmd)    │    │  ├─ muptiActivityEnable        │
│              │    │  ├─ muptiSubscribe              │
│  GPU 硬件调度  │    │  ├─ muptiProfilerBeginSession  │
│  内存管理     │    │  ├─ muptiPCSampling*           │
│              │    │  └─ 从 Driver 层截取性能计数器   │
└──────────────┘    └──────────────┬────────────────┘
                                   │ 回调 / 数据流
                                   ▼
                         ┌─────────────────────────┐
                         │ msight-compute          │
                         │  ├─ CLI: mcu            │
                         │  ├─ GUI: mcu-ui         │
                         │  ├─ capture/ → MUPTI    │
                         │  ├─ report/ → .mcu-rep  │
                         │  └─ 可视化: Timeline /  │
                         │     Roofline / PM采样   │
                         └─────────────────────────┘
```

### 4.2 调用链示意

```
(1) 应用调用 musaLaunchKernel()           [MUSA-Runtime]
(2) MUSA-Runtime → muLaunchKernel()       [musa Driver API]
(3) musa Driver → Core → HAL → M3D 调度   [musa Driver 内部]
(4) KMD 将 kernel 提交到 GPU 硬件          [KMD / GPU]
     ↑ 同时 MUPTI 在此注入回调
(5) MUPTI 回调 → msight-compute capture   [msight-compute]
(6) msight-compute 序列化为 .mcu-rep      [msight-compute]
```

### 4.3 职责划分

| 组件 | 角色 | 关键能力 |
|------|------|----------|
| **MUSA-Runtime** | 计算 API 门面 | CUDA Runtime 兼容，设备/Stream/Event/Graph 管理 |
| **musa Driver** | 驱动层 | 底层 GPU 控制、内存分配、SM 分区、ioctl 到 KMD |
| **MUPTI** | 剖析探针 | 性能计数器读取、kernel 活动记录、PC 采样、metric 收集 |
| **msight-compute** | 剖析消费端 | 采集编排、数据序列化、Timeline/Roofline 可视化 |

### 4.4 类比 NVIDIA 生态

```
MUSA-Runtime  ≈  CUDA Runtime (libcudart)
musa Driver   ≈  CUDA Driver API (libcuda)
MUPTI         ≈  CUPTI (CUDA Profiling Tools Interface)
msight-compute ≈  NVIDIA Nsight Compute (ncu / ncu-ui)
```

### 关注点

| 维度 | 说明 |
